# 子查询

子查询其实就是要在一条查询语句中再嵌套1条或多条查询语句
子查询可以出现的3个位置
	1、出现在where的后面作为一个或多个数据，
注意：这个子查询只能返回1列n行的数据，如果返回的是1列1行数据那么可以使用简单条件例如 = <  <= > >= !=
如果返回的是1列多行数据那么只能使用in 或not in
	2、出现在from后面作为一个临时表
注意：
	1、这个子查询可以返回n行n列的数据，
	2、这个子查询的显示列名作为临时表的字段名使用，
	3、如果子查询中使用了函数 那么必须要为这个显示列起别名
	4、出现在select后面作为显示列的一部分
注意：
	1、这个子查询需要返回1行1列的数据
	2、每一条记录都会执行一次这个子查询因此效率非常的低不推荐使用
注意：
	1、无论子查询出现在什么位置上，子查询必须要放到()里面

# 1.where中使用子查询

获取工资高于公司平均工资的所有员工
思路
	1、获取公司的平均工资
	select avg(sal) from emp
	2、使用步骤1的语句作为查询条件，获取工资高于步骤1查询结果的所有记录
	select * from emp where sal>(select avg(sal) from emp)

# 2.from中使用子查询

查询员工信息，查询哪些人是管理者，要求显示出其员工编号和员工姓名
思路
	1、获取所有的管理者的编号
	select mgr from emp where mgr is not null group by mgr
	2、使用步骤1的查询语句作为临时表temptable 然后使用emp与temptable进行多表查询
	使用emp中的empno与temptable表中的mgr消除笛卡尔积
	select * from emp e inner join temptable tt on e.empno=tt.mgr 
	3、使用步骤1的查询语句替换步骤2中的temptable
	select * from emp e inner join (select mgr from emp where mgr is not null group by mgr) tt on e.empno=tt.mgr

# 3.select中使用子查询

获取员工的信息以及部门的名称
思路
	1.获取所有的员工信息
	select * from emp e
	2.在步骤1的显示列中添加一个子查询，要根据当前记录行中的员工的部门编号作为条件查询dept表中的对应的部门名称
	select *,(select dname from dept where deptno=e.deptno) from emp e

# 查询结果合并union

将多条SQL语句的查询结果合并成1个结果
语法格式
	select语句1
	union [all]
	select语句2
	...
	union [all]
	select 语句n

注意：
	1、union 的 all是可选的表示获取多条语句中所有的数据内容包括重复的 默认表示消除重复 all的效率较高
	如果可以确定这些语句中没有重复的数据可以使用 union all
	2、这些语句中返回的列的数量必须要完全相等
	3、所有的语句中每个列的数据类型必须完全对相应相同
	4、所有的语句中每个列的含义也必须相同
	5、这个合并效率不高因此不推荐使用

# 分页查询limit

获取数据库表中的一部分的数据内容，用来提高数据库的响应速度，比较适合数据量大的时候使用
分页也可以提高用户的体验

语法格式
	select 显示列 from 表名 [where] [group by[having]][order by] limit 跳过的记录数量,获取多少条记录

跳过的记录数量 ：取值为正整数 ，大于等于0
获取多少条记录： 取值为正整数 ，大于0

# 5.获取前5条记录

select  * from emp limit 0,5
获取6到10的记录
select  * from emp limit 5,5
获取薪水最高的前5条记录
select  * from emp order by sal desc limit 0,5

# 创建表

语法格式
	create table 表名(  
	字段名1 数据类型[(长度)]  约束条件,
	字段名2 数据类型[(长度)]  约束条件,
	...
	字段名n 数据类型[(长度)]  约束条件
	)

常用的数据类型
	varchar(长度)	                变长字符串，存储空间等于实际数据空间
	double (有效数字位数，小数位)	数值型
	float(有效数字位数，小数位)	数值型
	Int( 长度)	                整型
	bigint(长度)	               长整型
	date	                       日期型 年月日
	datetime	               日期型 年月日 时分秒 毫秒

创建表t_student 拥有字段
	stu_no  字符串类型   长度64
	stu_name 字符串类型  长度64
	stu_age  数字       

	create table t_student(  
	stu_no varchar(64),
	stu_name varchar(64) ,
	stu_age int  
	);
注意：
	1.创建表示数字和日期类型不需要指定长度，字符串必须指定长度 ，浮点型必须指定长度
	2.字段名在当前表中不能重复
	3.表名在当前库中不能重复

# 修改表结构

语法格式  
	alter table 表名 add    字段名  数据类型[(长度)] 约束条件    向表中添加一个字段
	alter table 表名 modify 字段名 数据类型[(长度)] 约束条件   修改表中字段的类型 长度 以及约束
	alter table 表名 change 旧字段名 新字段名[(长度)] 约束条件  修改字段的名字 类型 长度 以及约束
	alter table 表名 drop   字段名  删除表中的字段同时删除数据

# 数据插入

语法格式
	全列数据插入，数据的类型以及含义必需与表中字段的默认顺序一一对应 不推荐
	insert into 表名 values(数据1,数据2...,数据n) 
	指定列插入 数据的类型以及含义必须与字段名的含义和类型一一对应 推荐
	insert into 表名(字段名1,字段名2...,字段名n)values(数据1,数据2...,数据n) 
注意
	1.插入数据时非数字型的字段数据必须使用'' 包括字符串和日期，而且日期必须要符合数据库的默认日期格式
	MySQL默认的日期格式为 yyyy-MM-dd 或 yyyy-MM-dd hh:mm:ss例如 '2019-03-26 15:21:20'

# 约束条件

常用的约束
	1、非空约束，not null  如果某个列添加非空约束那么在插入数据就必须要插入这个字段否则直接报错
	2、唯一约束，unique    如果某个列添加唯一约束那么这个列中的数据是不能重复的
	3、主键约束，primary key 如果某个列添加主键约束那么这个列不能为空也不能重复，主键通常不需要有任何的业务逻辑
	含义，作用是标记这条记录在表中的唯一性，通常我们会命名为 id bigint类型  自动递增
	4、外键约束，foreign key 如果某个列添加了外键约束那么这个列的值可以为空但是不能填写一个不存在的值，作用是
	   完成2个表之间的关联关系，例如学生中应该存放一个班级表中的主键作为外键，用于约束这个学生
	   是哪个班级下的

约束条件的使用语法有2中
	1.在创建表示添加
	2.在修改表结构时添加或修改或删除

# 数据的常用设计模式

	1.一对多 ：一个表中的一条记录要对应另外一个表的多条记录，例如一个班级下可以拥有多个学生
	实现方式： 在多的一方中定义个字段用于存放一的一方中的主键作为外键

	2.一对一：一表中的一条记录只能对应另外一张表中的一条记录，例如学生和学生的详细信息
	实现方式：在2个表任意的一个表中添加一个列用于存放另外一张表中的主键作为外键，并设置这个外键列为唯一约束
	推荐在详情表中添加外键
	3.多对多：2张表中的每条记录都会对应另外一张表中的多条记录，例如学生和课程
	实现方式：需要定义第三张表，这个表中只需要拥有2个列分别存放2个表中的id作为外键

	4.自关联一对多：当前表中的一条记录要对应当前表中的多条记录，例如商品的类型
	实现方式：在当前表中定义一个列用于存放当前表中某一条记录的主键值
