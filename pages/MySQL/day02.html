# 简单查询

查询select ，用于从数据表中获取指定的数据

# 语法格式

select 显示列名1,显示列名2 ,...显示列名n  from 表名
注意：
 1.显示列名默认与表中的字段名是一样的
 2.显示列可以直接进行数学计算 但要求这个字段的类型必须是数字型
 3.如果想获取所有的列那么可以使用 * 但是不推荐这么做，因为查询速度回降低，尽可能的需要哪些列就获取那些咧

# 练习01

查询所有的员工名字(ename)
 select ename from emp
查询所有的员工姓名(ename)以及职位(job)
 select ename,job from emp
查询所有员工的所有数据
 select * from emp
查询所有员工的名字以及年薪 （年薪=sal*12）
 select ename,sal*12 from emp

# 显示列设置别名

语法格式
 显示列名 as 别名
或
 显示列名  别名

注意：
 1.如果别名包含特殊字符例如空格 或汉字等等 必须要在别名上添加'' 例如 显示列名 as  '别名'

# 练习02

查询所有员工的名字以及年薪 （年薪=sal*12） 为年薪 添加别名
 select ename,sal*12 '年薪' from emp
 select ename,sal*12 as '年薪' from emp

# 条件查询

根据特定的条件获取数据库表中的数据例如获取性别为男的


语法格式
 select 显示列  from 表名 where 字段名 简单运算符 数据
注意：
 1.简单运算符包括 =  <  <=  >  >= !=
 2.数据库中所有的关键字不区分大小写包括表名和 字段名和库名， 但是数据时区分大小写的例如A 和 a就不是一
   个值可在MySQL中数据也不区分大小写
 3.在数据库中数字可以使用< <= > >= ，字符串也可以可以使用，字符串会按照字典顺序排列大小
 4.如果数据为非数字型则需要添加'' 例如字符串和日期格式数据

between ⋯ and ⋯ 运算符获取某个区间范围内容的数据
语法格式

注意：
 1.只能应用于数字字段
 2.它会包含开始值和结束值

and 和 or 运算符号
and 要求两侧的条件都真这条才会被获取 相当于Java中的 &&
or 要求两侧的条件有一个为真数据就会被获取  相当于Java中的||

语法格式
 字段名 = 数据 and 字段名 = 数据  ...
 字段名 = 数据 or  字段名 = 数据  ...

注意：
 1.由于and的优先级比or要高因此实际应用中可以根据业务的逻辑适当的添加()提升优先级

# 练习06

获取工资大于1600 并且小于等于3000的所有员工
 select * from emp where sal >1600 and sal<=3000
获取职位是办事员（CLERK） 或者 职位为 销售员（SALESMAN）的所有员工
 select * from emp where job='CLERK' or job='SALESMAN'
获取部门20和部门30中所有的办事员（CLERK）
 select * from emp where deptno=20 and job='CLERK' or deptno =30 and job='CLERK'
错误的：由于and的优先级比or要高所有SQL会先执行and 获取出部门30中的办事员及部门20中的所有员工
 select * from emp where deptno=20 or deptno=30  and job='CLERK'
正确的：提升or的优先级
 select * from emp where (deptno=20 or deptno=30)  and job='CLERK'


in 运算符  获取所有包含在某个范围内的所有数据

语法格式

注意：
 1.in的底层业务逻辑规则与or 是相同的
 2.由于or的运算速度比较慢，因此在实际应用时尽可的避免or和in的使用

# 排序order by

根据某一个或多列的数据进行降序或升序排序
语法格式
 select 显示列 from 表名 [where 条件] order by 字段名1 [排序规则],字段名2 [排序规则]...字段名n [排序规则]

注意：
 1.排序时可以根据一列或多个列同时排序，如果第一个列的值相同那么才会根据第二列排序
 2.可以对数字排序 ，可以对日期排序 日期越早值越小 ，如果越晚值越大，可以根据字符串排序它按照字典顺序排序
 3.排序规则，每个排序规则只能影响它前面一个字段，排序规则是可选的默认是升序。取值为 asc 升序、 desc降序

# 练习10

获取所有的员工信息根据工资降序
 select * from emp order by sal desc
获取所有的员工，根据部门编号升序后根据工资降序
 select * from emp order by deptno , sal desc
根据入职时间（hiredate）升序排序
 select * from emp order by hiredate

# 聚合函数

将多行中的某一列的值聚合成一行数据
 count(字段名)   根据字段名统计数量返回一个数字 也可将字段名替换成 * 但是效率不高
 sum(字段名)  将多行的指定列值相加求和 只能应用于数字列上
 avg(字段名) 将多行的指定列值相加然后取平局值 只能应用于数字列上
 max(字段名)  获取当前列的最大值可以应用在数字 字符串和日期以上  
 min(字段名) 获取当前列的最小值可以应用在数字 字符串和日期以上  
语法格式：
 select 函数名(字段名),函数名(字段名)...,函数名(字段名) from 表名 [where] [order by]
注意：
 1.所有的聚合函数都会自动的忽略掉为null的数据
 2.当显示列中使用了聚合函数以后那么现显示中不能出现非聚合的数据否则会出现错误，但是MySQL不会出现错误但是
   返回的数据是不正确的
 3.where 的后面不能直接使用聚合函数

# 分组查询 grop by-----------------------------------------

可以根据一个列或多个列同时进行分组 分组后会将这些列数据完全相同的行聚合成一行记录，因此显示列
中不能填写非分组数据或非聚合数据
语法格式
 select 显示列 from 表名 [where] group by 字段名1,字段名2...,字段名n [having 字段名 运算符 数据] [order by]

注意：
 1.分组后显示列中不能出现非分组数据和非聚合数据
 2.where 是分组前的条件筛选 having是分组后条件筛选 having中可以使用聚合函数
 3.分组后order by中可以使用聚合函数进行排序

# 练习12

获取各个部门中工资的平均值 显示部门编号和平均工资
 select deptno, avg(sal) from emp  group by deptno	
获取平均工资大于1500的所有职位以及职位的平均工资
 select job ,avg(sal) from emp group by job having avg(sal)>1500

获取各个部门中各个职位的平均工资，显示部门编号 职位  平均工资
 select deptno,job ,avg(sal)  from emp group by deptno,job 

获取部门30中各个职位的平均工资
 select job,avg(sal)  from  emp where deptno=30 group by job

# 连接查询 /多表查询

一条查询语句中拥有多个表，同时查询多个表中的数据，有2中语法 为92语法和99语法

# 92语法多表查询

语法格式
 select 显示列 from 表名1 [别名],表名2 [别名]  where 条件  [group by [having]] [order by]

# 练习13

获取所有的员工信息以及部门信息
 错误的：这条语句会出现笛卡尔积问题，笛卡尔积指多表查询时获取的记录内容是所有的表记录乘积，这表中所有的数据
        会逐一进行匹配，表越多数据越多那么获取的数据积也就越多
 select * from emp e,dept d
解决方案使用92语法
 需要在语句的后面添加一个where 的条件，我们需要利用这个条件获取对应的数据
 where条件的数量至少是表数量-1个，而且必须使用多表之间的关联列消除笛卡尔积例如emp和dept表中的deptno就是关联列

select * from emp e,dept d  where e.deptno=d.deptno
注意：
 1.在多表查询时推荐为表名添加别名然后使用别名.字段名 ，这是为了防止这些表中会出现同名的字段名

# 99语法实现多表查询

语法格式
内连接
 select 显示列 from 表名1 别名 inner join 表名2 别名 on 条件 inner join 表名3 别名 on 条件 ... [where] [group by[having]] [order by]
左外连接 
 select 显示列 from 表名1 别名 left outer join 表名2 别名 on 条件 left outer join 表名3 别名 on 条件 ... [where] [group by[having]] [order by] 
右外连接 
 select 显示列 from 表名1 别名 right outer join 表名2 别名 on 条件 right outer join 表名3 别名 on 条件 ... [where] [group by[having]] [order by] 

注意：
 1. inner 和 outer 可以省略不写
 2.99语法的内连接获取的数据与92语法是完全一样
 3.左外连接和右外连接是相互的作用是当前2个表的数据不能完全匹配时，
   如果需要让左侧的表多显示数据那就用用左外否则就是用右外
 4.on 后面的条件可以使用and 或or 同时添加多个

# 练习14

获取所有的员工信息以及部门信息
 select * from emp e inner join dept d on e.deptno=d.deptno    
或
 select * from emp e  join dept d on e.deptno=d.deptno   

获取所有员工信息和部门信息同时显示没有员工的部门信息
右外连接
 select * from emp e right outer join dept d on e.deptno=d.deptno
或
 select * from emp e right  join dept d on e.deptno=d.deptno

左外连接
 select * from  dept d left outer join emp e on e.deptno=d.deptno 
或
 select * from  dept d left  join emp e on e.deptno=d.deptno
